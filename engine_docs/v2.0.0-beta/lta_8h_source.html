15 /*
16  Using shading normal for light transport algorithms is equivalent to using
17  asymmetric BSDFs. This can lead to inconsistent results between regular
18  and adjoint algorithms. This correction factor, when multiplied with BSDF,
19  restores the symmetric condition of the BSDF during importance transport.
20 
21  <Ns>: shading normal
22  <Ng>: geometry normal
23  <L>: direction of incident importance
24  <V>: direction of excitant importance
25 
26  Note that all vectors are expected to be in unit length and leaving
27  the surface.
28 
29  References:
30 
31  Eric Veach, "Non-symmetric Scattering in Light Transport Algorithms",
32  Eurographics Rendering Workshop 1996 Proceedings.
33 */
34 inline real importance_BSDF_Ns_corrector(
35  const Vector3R& Ns,
36  const Vector3R& Ng,
37  const Vector3R& L,
38  const Vector3R& V)
39 {
40  PH_ASSERT_GT(std::abs(Ns.dot(V) * Ng.dot(L)), 0.0_r);
41 
42  return std::abs((Ns.dot(V) * Ng.dot(L)) / (Ng.dot(V) * Ns.dot(L)));
43 }
